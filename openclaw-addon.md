化策略

从公开材料看，OpenClaw 还没有把这些作为主产品抽象公开出来。

一个很有意思的旁证是 ResonantOS。它明确说自己是运行在 OpenClaw 之上的“experience layer”，甚至直接用了“OpenClaw is the kernel, ResonantOS adds the intelligence”这种说法。这反而支持了我们的判断：社区里已经有人把 OpenClaw 当成底层运行内核，而把更高层的 intelligence / orchestration 放到上层系统里。

所以如果你让我一句话定位：

OpenClaw 当前更像“AI assistant runtime + gateway/control plane”，不是完整的 Work OS；它有局部 intelligence，但还不是组织级的 intelligence layer。

如果套到我们前面的术语里，大概是这样：

OpenClaw ≈ Gateway + Agent Runtime + Node Interface

还缺 ≈ Work Definition + Capability Graph + Orchestration Intelligence + Evaluation System

ResonantOS 这类上层项目，开始往 Intelligence / Experience / Orchestration Layer 走。
